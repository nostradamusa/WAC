# AI Development Rules

Always follow these rules when modifying the codebase.

1. Prefer full file replacements instead of partial snippets.

2. Do not introduce new dependencies unless necessary.

3. Do not rewrite working features unless requested.

4. Maintain compatibility with Supabase schema.

5. Do not convert the project to a different architecture.

6. Avoid monolithic files larger than 500 lines.

7. Keep UI components modular.

8. Preserve current database naming conventions.
